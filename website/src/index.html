Everything citation tools should be

One-click citing

Click the Ibid icon on any page. Metadata auto-fills from structured data. Copy the formatted citation instantly.

Smart auto-enhance

Paste a DOI, ISBN, PMID, or arXiv ID. Ibid resolves it via CrossRef, Open Library, or NCBI and fills missing fields automatically.

Actually accurate

Powered by Hayagriva — the same CSL engine that powers Typst. Supports all 2,600+ official Citation Style Language styles with correct et al., title case, and locale terms.

Zero data collection

No telemetry. No analytics. No bandwidth hijacking. All processing happens locally in your browser via Rust/WebAssembly.

Import & export anything

Import from BibTeX, RIS, CSL-JSON, EndNote XML, MEDLINE, CSV. Export to 9 formats. Drag & drop files or paste text. Filter exports by project and type.

Project-based library

Organize citations into projects with per-project styles. Sort, filter, star, search, bulk operations. Side panel stays open while you research.

Why Ibid?

Built because existing citation tools get formatting wrong, send your data to servers, or lock features behind paywalls.

🔒
Your data stays with you All citation processing runs locally via WebAssembly. No page data, browsing history, or citation content is ever sent to any server.
✅
Accurate formatting Powered by Hayagriva — the CSL engine from the Typst project. Handles et al., title case, locale terms, and page ranges correctly across all 2,600+ styles.
🔓
No account, no paywall Every feature works immediately. Import, export, projects, tags, 2,600+ styles — all free. No signup, no premium tier, no limits.
🛡
Minimal permissions Only requests access to the page you're on. No host permissions, no browsing history, no background page access. Your browser stays yours.
🔌
Works offline 74 citation styles bundled. Citation formatting, library management, import/export — all work without an internet connection. DOI lookups need network.
⚙
Rust/WebAssembly engine Hayagriva + citationberg compiled to WASM. BibTeX, RIS, EndNote XML, MEDLINE, CSV parsed natively. No server, no delays.
